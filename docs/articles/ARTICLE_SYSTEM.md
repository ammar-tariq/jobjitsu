Create docs/articles/ARTICLE_SYSTEM.md.

Document the JobJitsu article management system.

The system should define:

- Why we write articles
- When a new article should be created
- How AI detects article-worthy milestones
- Article approval workflow
- Article structure
- Community invitation section
- Historical record keeping

Rules:

Articles should only be created for meaningful milestones.

Examples:

Create article when:

- New architecture milestone is completed
- Major feature is released
- New AI agent capability is introduced
- Major engineering challenge is solved
- Important open-source milestone happens
- Community contribution changes project direction

Do not create articles for:

- Bug fixes
- Dependency updates
- Small refactors
- Minor documentation changes

Do not write the article. Only define the system.
