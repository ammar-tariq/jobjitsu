## Principles

### 1. Privacy is the platform
On-device by default. Outbound only when the user acts. Visible **Agent · On-device** trust chrome is part of the product, not a footnote.

### 2. Sovereignty over convenience theater
User owns preferences, pause, approve, and send. Convenience never removes the signal.

### 3. Leverage over volume
Optimize for well-aimed throws and fit — not vanity send counts or “50 today” quotas.

### 4. One calm operating system
Modules share voice, visual language, and mental model. No cockpit sprawl; one job per view.

### 5. Agent as belt, not leash
Automation increases readiness and technique. It does not replace judgment or invent urgency.

### 6. Craft over choreography
Celebrate consistency and skill. Treat setbacks as information. No confetti for routine saves; no shame for rest.

### 7. Open trust
Open-source inspectability and local execution beat marketing promises.

### 8. Native, light, fast
Desktop tool energy: crisp, keyboard-capable, respectful of the machine. Dark mode first; Midnight + teal identity holds.

### 9. Honest AI
Help draft, tailor, queue, and remind. Never claim hiring outcomes the product cannot promise.

### 10. Brand as product behavior
If it wouldn’t fit calm confidence — Leverage Belt, teal flap sacred, metaphor as seasoning — it doesn’t ship as JobJitsu.

## Principle checklist

Use before committing roadmap or module work:

| Ask | Must answer |
|-----|-------------|
| Does this keep data on-device by default? | Yes, or redesign |
| Does this increase leverage or only volume? | Leverage |
| Does the user retain control of outbound actions? | Always |
| Does this reduce anxiety or manufacture it? | Reduce |
| Does this deepen the Career OS or distract into a side quest? | Deepen |
| Would a black belt say this calmly? | If no, rewrite or cut |
| Does it violate a [non-goal](./NON_GOALS.md)? | If yes, stop |

## Relationship to brand

Product principles operationalize brand philosophy. Copy, motion, and empty states must still follow [../brand/VOICE_AND_TONE.md](../brand/VOICE_AND_TONE.md) and [../brand/DESIGN_SYSTEM.md](../brand/DESIGN_SYSTEM.md). Principles decide *whether* something belongs; brand decides *how it feels*.
